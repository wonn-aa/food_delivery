# food_delivery
srez_1

Отчет по лабораторной работе: Работа с Apache Kafka

Цель работы: Развертывание Apache Kafka в режиме KRaft, разработка Producer и Consumer на языке Python для имитации работы службы доставки еды.
1. Описание выполненных действий

Шаг 1: Развертывание инфраструктуры
Использована версия Apache Kafka 2.13-4.2.0.
В связи с отсутствием утилиты wmic в системе, была настроена переменная окружения $env:KAFKA_HEAP_OPTS="-Xmx1G -Xms1G" для корректного выделения памяти.
Выполнено форматирование хранилища метаданных KRaft с уникальным ID кластера:
.\bin\windows\kafka-storage.bat format --standalone -t fBcW0-Z0Rqe_m0GXx6fU4Q -c .\config\kraft\server.properties
Запущен сервер Kafka через конфигурацию KRaft (без использования ZooKeeper).

Шаг 2: Разработка скриптов
data_gen.py: Реализована функция generate_order(), генерирующая JSON-объекты (ресторан, меню, клиент, курьер, рейтинг). Соблюден принцип SOLID (логика генерации отделена от логики отправки).
producer.py: Скрипт подключается к Kafka, вызывает генератор и отправляет сообщения в топик food_delivery с использованием метода flush() для мгновенной доставки.
consumer.py: Скрипт подписывается на топик, проводит валидацию (рейтинг заказа должен быть выше 0). В случае невалидных данных выводит сообщение с пометкой NOT VALID.

Шаг 3: Тестирование
Создан топик food_delivery.
Запущены параллельно процессы Consumer и Producer.
Проверена передача сообщений в режиме реального времени.

3. Инструкция по запуску

a) Установить зависимости: pip install confluent-kafka.

b) Запустить сервер Kafka (KRaft).

c) В отдельном окне запустить потребителя: python consumer.py.

d) В отдельном окне запустить поставщика: python producer.py.
